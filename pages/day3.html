Directives

What is a Directive and what to use for?

 

Essentially, a directive is a function which is executed when the AngularJS compiler finds it in the DOM. Such function can do almost anything you want to use it for, so the desition depends on what you have to achieve and your design. But, basically you use for encapsulating a proccess which you want keep apart from the main flow or for doing repetitive tasks.

There are three types of directives: framework's native, from third parties and custom. The convention is that the angular's native directives always use the prefix ng-.

Every directive has a name (like ng-repeat, tabs, make-up-your-own) and each directive determines where it can be used: element, attribute, class, in a comment.

Besides when you use a third party one, it must to be injected to the controller's function which will use it.

Check out the Angular's Directive documentation AngularJs Developer Guide: Directives

Services

What a Service is useful for?

 

AngularJS services are singleton objects whose objective is to be reusable independently of the views. It acts pretty much as a Constructor Function and even can added public methods to it.

It holds a reference to an object which will return a response to the control trough the service.

There are native AngularJs services, some of them were used in the previous exercises, such as: $http & $q, $timeout, $interval, $provide ( for advanced AngularJs Modules), $route, $location, $window, etc. They can be recognized because they ussually use the $ sign as a prefix.

Some of them are extensions of the Vanilla Javascript methods but using AngularJS services we assure the interaction with the properties, variables and objects of the $scope.

There are also custom services and, like the natives, they must be injected to the main controller.

Check out the Angular's Services documentation AngularJs Developer Guide: Services & AngularJs Services www.w3schools.com

Factories

What is a Factory and what to use for?

 

A Factory can give the same capabilities of a service but, as its name tells, working as the Factory Pattern Design.

It can be used to create almost any things we need in our design, that fact reduces the Factories types to the custom created by our own.

The first question we have to do ourselves is what type of object do we need? We can create complex objects, even such as new Class instances (with .prototype or ES2015 Classes), functions or even just return a simply string. You can create whatever you like, that’s the rule.

As the Directives and Services, the Factories must be injected to the controller which will use it, so they will be available when are invoked.

 

Check out the Angular's Factories documentation AngularJs Developer Guide: Factories

Recap

$scope

  • The scope is the binding way between the HTML (view) and the JavaScript (controller).
  • Is an object with the available properties and methods and they are is available for both the view and the controller.

 

Controllers

  • They are defined as a JavaScript object with $scope as argument.
  • We can have so many controllers as we need, but the scope of any of them is limited to the referenced DOM's element.

 

Modules

  • Helps to name, reference and group services, directives, filters, configuration information and initialization code.
  • Make possible avoid global variables and control the dependencies injection.

 

Directives

  • Are responsible for updating the DOM when the state of the model changes.
  • Extend HTML functionallity.
  • Angular comes with a built in set of directives (ng-*) which are useful for building web applications, we can create directives of our own too or import them from third parties.

 

Services

  • Are singletons functions, so there is only one instance of each service you define.
  • As singletons, they are not affected by scopes and hence can be referenced by multiple views/controllers/directives/other services.
  • They are created when two or more things need access to the same data or when you want to encapsulate interactions with, lets say, a web server.

 

Factories

  • They are function which return a created object on the go.
  • Their code can be reused for the controller wheir they are nested.
  • The object created doen't exist until is invoked and as any on object can be updated or removed.
